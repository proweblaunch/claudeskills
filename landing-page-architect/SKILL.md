# Landing Page Architect

## Mission

Design landing pages that convert by combining clarity, persuasion, and premium presentation.

A landing page should not feel like a generic sequence of marketing blocks.

It should feel like a carefully engineered argument that leads the visitor to action.

## What this skill is for

Use this skill when the user wants help with:

- landing page structure
- hero section direction
- conversion-focused page sequencing
- premium lead generation pages
- product launch pages
- service marketing pages
- scroll narrative for a single goal
- persuasive homepage-like landing flows

## When to use the skill

Use this skill when the goal is to:

- get signups
- drive inquiries
- book calls
- promote a product or service
- explain an offer and persuade the visitor to act

## When not to use the skill

Do not use this skill when the page is primarily:

- an editorial article
- a dashboard
- a documentation page
- a purely internal application screen
- a simple informational reference page with no conversion goal

## Core architecture principle

A landing page should create a smooth decision path.

That path usually moves through:

- attention
- relevance
- understanding
- desire
- trust
- action

The visitor should always know what the page is trying to do.

## Hero section logic

The hero must answer quickly:

- what is this?
- who is it for?
- why does it matter?
- what should I do next?

The hero should not waste space on vague branding if the user needs clarity.

The best hero sections combine:

- strong headline
- supporting clarity
- visual proof or product representation
- immediate action pathway

## Section sequencing

The page should usually move through:

1. hook
2. problem or opportunity
3. solution or offer
4. proof or trust
5. features or benefits
6. differentiation
7. social proof or credibility
8. final conversion

This sequence can be adapted, but the logic should remain persuasive.

## Persuasion principles

A good landing page should reduce hesitation.

It should make the offer feel:

- clear
- credible
- relevant
- beneficial
- easy to understand
- worth exploring

## Design principles

The design should support conversion through:

- clean hierarchy
- intentional spacing
- easy scanning
- strong CTA placement
- clear visual anchors
- controlled motion
- premium feel without clutter

## CTA strategy

Calls to action should feel like the next logical step.

Use them when the page has done enough work to earn the ask.

Avoid overusing CTAs too early.

## Trust strategy

Use evidence where needed:

- testimonials
- results
- logos
- case studies
- credentials
- process clarity
- guarantees or assurances
- product proof

Trust should be integrated into the page rather than tacked on at the end.

## Motion strategy

Motion should help guide the eye and reinforce confidence.

Use it to:

- reveal sections smoothly
- emphasize the most important content
- create a premium feel
- reduce visual abruptness

Avoid motion that competes with the message.

## Quality standard

A strong landing page should be:

- easy to understand
- difficult to ignore
- persuasive without feeling pushy
- premium without feeling overdesigned
- structured to reduce confusion and increase confidence

If the page feels busy, simplify the message.
If the page feels weak, sharpen the promise.
If the page feels generic, make the offer more specific.
