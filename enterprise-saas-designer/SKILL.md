# Enterprise SaaS Designer

## Mission

Design websites for enterprise software and premium SaaS brands that must feel clear, trustworthy, modern, and strategically positioned.

This skill should help Claude think like a designer for software companies selling capability, reliability, scale, and confidence.

## What this skill is for

Use this skill when the user wants help with:

- SaaS homepage direction
- enterprise software marketing pages
- product storytelling for software platforms
- trust-building website structure
- business software positioning
- feature presentation and capability explanation
- visual direction for modern software brands
- enterprise-focused conversion design

## When to use the skill

Use this skill when the request includes:

- SaaS website
- software platform
- enterprise product
- dashboard product marketing
- B2B software messaging
- product-led website structure
- premium software visuals
- software startup or scale-up branding

## When not to use the skill

Do not use this skill when the request is about:

- a purely artistic portfolio site
- an e-commerce store with no SaaS context
- a personal blog or non-software brand
- a basic code fix without product-design implications

## Core design philosophy

Enterprise software websites must earn attention through clarity, trust, and structure.

They should feel:

- intelligent
- controlled
- modern
- credible
- scalable
- efficient
- secure
- premium without being flashy

The design should not rely on hype.

It should communicate competence.

## Primary goals

A strong enterprise SaaS site should help the visitor understand:

- what the product does
- who it is for
- what problem it solves
- how it differs from alternatives
- why it can be trusted
- why it can scale
- why it is worth exploring further

## Layout principles

Use a structure that supports fast understanding.

Typical strong patterns include:

- clear hero with precise value proposition
- proof or credibility section early
- feature groups organized by outcome
- workflow or process explanation
- customer evidence or trust signals
- pricing or plan clarity where relevant
- strong conversion path

## Messaging principles

Enterprise SaaS messaging should be:

- precise
- confident
- uncluttered
- useful
- outcome-focused
- credibility-driven

Avoid vague claims.
Avoid inflated marketing language.
Avoid overpromising.

## Visual direction

A premium SaaS site often benefits from:

- high contrast structure
- strong hierarchy
- modern typography
- clean spacing
- subtle gradients
- measured use of glass or translucency
- restrained motion
- product UI framing
- data visualization style visuals
- crisp product illustration or interface imagery

## Trust signals

Enterprise buyers need reassurance.

The site should make room for:

- security
- compliance
- scale
- reliability
- support
- implementation maturity
- product depth
- integrations
- case studies or outcomes
- technical seriousness

## Storytelling model

Use a narrative that moves from:

- attention
- clarity
- usefulness
- proof
- confidence
- action

The page should feel like a guided product evaluation, not a generic marketing page.

## Product presentation

Present the software in terms of:

- user outcomes
- workflows
- business value
- system depth
- differentiation
- operational benefit

Do not only list features.

Show why the feature matters.

## Motion and interaction

Motion should support understanding, not distract from it.

Use motion to:

- reveal sections with polish
- emphasize product screenshots
- move between product ideas
- support hierarchy
- create a feeling of precision

Avoid motion that looks overly playful or consumer-gimmicky.

## Quality standard

A good enterprise SaaS design feels:

- clear under pressure
- trustworthy at a glance
- confident without arrogance
- modern without being trendy
- premium without being noisy
- structured enough for serious buyers

If the experience feels too vague, make the value proposition sharper.
If it feels too decorative, make it more useful.
If it feels too generic, make it more specific to the product and audience.
