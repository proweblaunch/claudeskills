# Design System Architect

## Mission

Define a reusable design system that makes a website or product feel coherent, scalable, and premium.

A design system is not just a set of components.
It is a shared language for spacing, typography, colors, states, hierarchy, and interaction behavior.

## What this skill is for

Use this skill when the user wants help with:

- design system structure
- component consistency
- spacing and layout rules
- shared UI patterns
- scalable visual standards
- premium interface consistency
- multi-page website cohesion
- product UI foundations

## When to use the skill

Use this skill when the interface needs to feel unified across multiple screens or sections.

## When not to use the skill

Do not use this skill when the request is only about a single isolated visual element with no system implications.

## Core philosophy

A strong design system reduces randomness.

It helps the entire experience feel like it came from one disciplined team.

Premium design systems usually prioritize:

- consistency
- predictability
- clarity
- flexibility
- visual discipline
- maintainability
- brand coherence

## System responsibilities

This skill should think about:

- color rules
- spacing scales
- typography hierarchy
- button styles
- card patterns
- section spacing
- surface treatment
- icon usage
- motion standards
- layout behavior
- responsive adaptations
- state definitions

## Consistency logic

Consistency should not mean monotony.

A good system allows variation while preserving identity.

The goal is to make every part of the interface feel related.

## Premium system quality

A premium system often has:

- clear tokens or standards
- strong spacing rhythm
- restrained color usage
- consistent radii and shadows
- clear emphasis rules
- reusable content patterns
- predictable responsive behavior

## Output behavior

When using this skill, focus on rules that support scale and quality.

If a design choice cannot be reused or explained consistently, it may not belong in the system.
