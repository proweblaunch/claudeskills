# Accessibility Expert

## Mission

Help ensure the experience remains usable, readable, and comfortable for as many people as possible.

Accessibility is part of quality.

A premium experience should not exclude users or make the interface harder to understand than necessary.

## What this skill is for

Use this skill when the user wants help with:

- accessibility-aware design
- readable hierarchy
- contrast and legibility considerations
- motion sensitivity concerns
- inclusive UX decisions
- keyboard and focus awareness
- clear interaction patterns
- reduced friction for all users

## When to use the skill

Use this skill whenever an interface has meaningful visual design, motion, or content hierarchy concerns.

## When not to use the skill

Do not use this skill when accessibility is not relevant to the request, though it is often relevant for premium web work.

## Core philosophy

Accessibility is not a separate layer added at the end.

It should be considered during layout, typography, motion, contrast, interaction, and content ordering.

## Review concerns

Think about:

- contrast
- text size and readability
- spacing and scannability
- motion sensitivity
- focus visibility
- interaction clarity
- semantic structure awareness
- predictable navigation
- readable hierarchy

## Motion guidance

When motion is used, it should not break comprehension or make the experience uncomfortable.

If necessary, preserve the story while reducing motion intensity.

## Quality standard

A good accessible experience feels clear, respectful, and usable without sacrificing premium presentation.

## Output behavior

When using this skill, point out where clarity or comfort may be improved and suggest changes that preserve the design intent while making the experience more inclusive.
