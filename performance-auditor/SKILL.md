# Performance Auditor

## Mission

Review the experience through the lens of speed, smoothness, and practical delivery.

A premium website should feel fast, responsive, and stable.

This skill helps Claude judge whether a design can survive real-world device, network, and rendering constraints.

## What this skill is for

Use this skill when the user wants help with:

- performance review
- animation cost awareness
- frontend responsiveness
- asset weight concerns
- rendering bottlenecks
- motion-heavy website feasibility
- smoothness and polish under real conditions

## When to use the skill

Use this skill when the user is designing anything ambitious, especially if motion, 3D, or visual effects are involved.

## When not to use the skill

Do not use this skill when the request is purely creative and performance is not a concern.

## Core philosophy

A beautiful experience that feels slow is not premium.

Performance is part of perception.

The interface should feel smooth, controlled, and trustworthy.

## Audit concerns

Think about:

- animation cost
- excessive repaint or reflow risk
- overuse of heavy effects
- asset weight
- image and video strategy
- device limitations
- interaction smoothness
- page load behavior
- mobile behavior

## Quality standard

A good performance-aware design keeps the premium feeling while avoiding waste.

If a visual idea is too expensive for the target audience’s devices, reduce complexity rather than sacrificing quality entirely.

## Output behavior

When using this skill, identify where the experience might feel heavy, what would likely be most expensive, and what simplifications would preserve the experience best.
