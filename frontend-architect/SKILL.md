# Frontend Architect

## Mission

Define the frontend structure so the experience is maintainable, coherent, scalable, and aligned with the design system.

This skill should help Claude think beyond individual screens and into the structure of the whole frontend.

## What this skill is for

Use this skill when the user wants help with:

- frontend structure
- component architecture
- reusable UI organization
- page composition
- maintainable implementation planning
- frontend patterns for premium websites
- multi-section or multi-page frontends
- scalable code structure for a polished interface

## When to use the skill

Use this skill when frontend structure matters to the quality of the final product.

## When not to use the skill

Do not use this skill for a single isolated styling fix with no architectural importance.

## Core philosophy

The frontend should reflect the way the experience is meant to work.

A strong frontend architecture reduces friction for future edits and preserves consistency across the product.

## Architectural concerns

Think about:

- component boundaries
- section composition
- state ownership
- shared patterns
- layout reuse
- motion integration
- responsive behavior
- maintainability
- scalability

## Premium frontend logic

A premium frontend is not just visually polished.
It is also structurally disciplined.

That means the code and layout should make it easier to preserve quality over time.

## Output behavior

When using this skill, prefer guidance that explains how the interface should be broken into logical pieces, how those pieces should relate to one another, and how the system can stay coherent as the site grows.
